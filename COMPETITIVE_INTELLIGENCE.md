### 1. Market Coordination Signals
**Pattern:** Multiple competitors' APIs change simultaneously

**What it means:** Coordinated response to market event

**Pattern:** Response times spike, error rates increase, gateway changes

**What it means:** Competitor is under stress (DDoS, outage, rapid scaling)

**Example:** Detect when competitor's API response times increase 10x, indicating infrastructure failure or attack

**Value:** Timing signal for market moves

### 3. Algorithm Change Signals
**Pattern:** API response patterns change, new endpoints appear, rate limiting changes

**What it means:** Competitor is deploying new features or changes

**Example:** Detect when competitor's API adds new endpoints 48 hours before announcement

**Value:** Early warning of competitive moves

### 4. Regulatory Signals
**Pattern:** Infrastructure changes correlate with regulatory announcements

**What it means:** Competitor is responding to regulatory pressure

**Example:** Detect when competitor's infrastructure changes after SEC filing

**Value:** Proof of regulatory impact

### 5. M&A Signals
**Pattern:** Infrastructure consolidation, new data center activation, bandwidth spikes

**What it means:** Acquisition or merger activity

**Example:** Detect when competitor activates new data center infrastructure weeks before M&A announcement

**Value:** Early warning of market-moving events
